# TripTracer v0.0.1 Bootstrap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 TripTracer 创建 monorepo 目录骨架、占位容器文件与可直接用于预拉基础镜像的 `docker-compose.yml` 基线。

**Architecture:** 根目录维护基础设施编排与环境变量模板，应用代码按 `frontend`、`backend`、`algo` 三个子目录拆分。`v0.0.1` 只建立可运行的结构与中间件编排，不实现前后端业务代码。

**Tech Stack:** Docker Compose, Dockerfiles, Markdown, dotenv conventions

---

- [ ] **Step 3: 写入 `.gitignore` 与 `.env.example`**

`.gitignore` 需覆盖：

```gitignore
.env
.env.*
!.env.example
__pycache__/
*.pyc
.pytest_cache/
.venv/
node_modules/
dist/
build/
.DS_Store
```

`.env.example` 需包含：

```env
TRIPTRACER_MYSQL_DATABASE=triptracer
TRIPTRACER_MYSQL_USER=triptracer
TRIPTRACER_MYSQL_PASSWORD=triptracer_dev_password
TRIPTRACER_MYSQL_ROOT_PASSWORD=triptracer_root_password
TRIPTRACER_ELASTIC_PASSWORD=triptracer_es_password
MINIO_ROOT_USER=triptracer
MINIO_ROOT_PASSWORD=triptracer_minio_password
```

- [ ] **Step 4: 写入 `docker-compose.yml`**

`docker-compose.yml` 需包含 `mysql`、`elasticsearch`、`redis`、`minio` 四个服务、持久化卷、健康检查与端口映射，并引用 `.env` 约定变量。
